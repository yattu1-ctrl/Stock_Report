# Codex 作業ルール

## 基本方針
- 作業開始前に git の状態とブランチを確認する。
- 機能単位や大きなステップ完了時に、変更内容が分かる簡潔な日本語メッセージでコミットする。
- 既存の変更を勝手に戻さず、ユーザーの作業内容を尊重する。
- 作業対象は `/Users/azmacbook/Projects/Stock_Report` とする。

## Git コミットルール
- チェックポイントごとに git コミットを行い、随時進捗を保存する。
- コミットメッセージは変更内容が分かる簡潔な日本語で記載する。
- 作業開始前にブランチが適切か確認し、必要に応じて `codex/` 接頭辞の新しいブランチを作成する。

## セキュリティルール
- API キー、パスワード、トークン、秘密鍵などの機密情報をリポジトリへコミットしない。
- `.env` と `.env.*` は追跡対象外とし、共有が必要な場合は値を空にした `.env.example` を使う。
- 外部スクリプトや外部 CDN を追加する場合は、必要性を確認し、信頼できる提供元に限定する。
- フォームを実装する場合は、個人情報の取り扱い範囲を明確にし、不要な情報を収集しない。
- ユーザー入力を画面に表示する実装では、HTML として直接挿入せず、エスケープまたは安全な DOM API を使う。
- 生成物や一時ファイルに個人情報・機密情報が含まれる可能性がある場合は、保存場所と追跡対象を確認する。
- 投資関連の表示では、断定的な売買推奨を避け、判断材料・確認項目・リスクを明示する。

## レポート制作メモ
- 国内株式のデータセンター関連、半導体関連を主な対象とする。
- 参照元・時点・前提条件を明記し、情報更新が必要な箇所を分かりやすく残す。
- 依存関係を追加する場合は、目的と影響範囲を確認してから行う。
